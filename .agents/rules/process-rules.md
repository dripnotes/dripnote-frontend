# 프로세스 규칙 (Process Rules)

- AI 에이전트는 `.agents/workflows/vibe-coding-cycle.md`에 정의된 **6단계 생명주기를 절대적으로 준수**해야 합니다.
- **2단계(허가/Approval) 묵살 금지**: 사용자가 자동 실행을 명시적으로 허락했거나 구조에 영향을 주지 않는 단순한 수정이 아닌 이상, 초기 프롬프트만 듣고 구두 승인 없이 즉시 애플리케이션 코드를 수정해서는 안 됩니다.
- **항상 문서 업데이트하기**: 성공적인 구현 주기(6단계)가 끝날 때마다, 반드시 `specs/` 디렉토리에 있는 해당 기능 명세서를 최신화하거나 구현 완료 상태로 반영해야 합니다. 또한, **커밋 전 반드시 CHANGELOG.md와 히스토리 아카이브를 작성/업데이트**해야 합니다.
- **PLAN.md 의무 작성**: 1단계(계획) 진행 시, 반드시 프로젝트 최상단 루트에 `PLAN.md` 파일을 생성 또는 갱신하여 사용자가 작업 환경에서 구체적인 계획안을 직접 읽을 수 있도록 조치해야 합니다.
- **한국어 커밋 메시지**: 모든 Git 커밋 메시지는 한국어로 작성하며, 작업 단위를 명확히 구분하여 기술합니다.

- **외부 AI 리포트 대응**: 사용자가 'Prompt for AI Agents' 형식의 리포트 조각을 제공하는 경우 전적으로 `.agents/workflows/ai-review-lifecycle.md`의 워크플로우에 따라 수행해야 합니다. 분석 후 한국어로 설명하고, 최종 승인 시 한국어로 커밋 메시지를 생성하여 커밋까지 완료합니다.
