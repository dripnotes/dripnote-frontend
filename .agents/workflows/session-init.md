# 세션 초기화 워크플로우 (Session Initialization)

이 워크플로우는 AI 에이전트가 새로운 대화 세션을 시작할 때 프로젝트의 특수 규칙과 현재 진행 상황을 완벽하게 파악하도록 강제하기 위해 존재합니다.

## 🤖 에이전트 체크리스트

### 1단계: 전역 규칙 및 프로세스 재확인

- [ ] `.agents/rules/global.md`를 읽고 프로젝트의 14번(문서 수정 사전 검토제), 21번(변경 사유 명시) 규칙을 상기하십시오.
- [ ] `.agents/rules/process-rules.md`를 읽고 `PLAN.md` 작성 의무를 확인하십시오.

### 2단계: 현재 진행 상황 동기화

- [ ] 프로젝트 루트의 `PLAN.md`를 읽어 현재 어떤 작업이 진행 중인지, 다음 단계가 무엇인지 파악하십시오.
- [ ] `task.md` (진행 중인 경우)를 확인하여 체크리스트의 상태를 업데이트하십시오.

### 3단계: 환경 확인

- [ ] `apps/web/.env.local` 파일의 존재 여부와 필요한 환경 변수 설정을 확인하십시오 (백엔드 URL 등).
- [ ] 현재 구동 중인 터미널 세션이 있다면 `pnpm run dev` 등의 상태를 체크하십시오.

## 실행 지침

모든 에이전트는 작업을 시작하기 전 "세션 초기화 완료: 현재 규칙과 PLAN.md를 확인했습니다."라고 사용자에게 보고한 뒤 첫 번째 태스크를 수행해야 합니다.
