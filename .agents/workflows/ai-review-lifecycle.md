# AI 리포트 자동 대응 워크플로우 (AI Review Lifecycle)

사용자가 'Prompt for AI Agents' 형식의 리포트 조각을 제공했을 때, AI 에이전트는 아래의 자동화된 절차를 따라 대응합니다.

## 1. 리포트 분석 (Analyze Report)

- 복사된 리포트 내용을 분석하여 현재 코드베이스의 어느 파일, 어느 라인에 해당하는지 매핑합니다.
- 리포트의 내용이 유효한지(버그인지, 개선 사항인지) 판단합니다.

## 2. PLAN.md 계획 제안 (Plan & Propose)

- 분석한 내용을 바탕으로 프로젝트 루트의 `PLAN.md`에 수정 계획을 작성합니다.
- 사용자에게 "리포트를 분석한 결과 다음과 같은 수정이 필요해 보입니다. 진행할까요?"라고 승인을 구합니다.

## 3. 구현 및 기술적 설명 (Implement & Explain)

- 승인 시 즉시 코드를 수정하고, 수정한 이유와 기술적 세부 사항을 한국어로 개발자에게 설명합니다.
- `ui-library` 연동이나 타입 안정성 등 하네스 엔지니어링 룰을 준수하며 구현합니다.

## 4. 사용자 최종 리뷰 (Final User Review)

- 구현된 결과물을 개발자가 직접 확인하도록 안내합니다.
- 레이아웃이나 상세 로직에 대한 피드백이 있다면 이를 반영(Polishing)합니다.

## 5. 한국어 자동 커밋 (Auto Korean Commit)

- 사용자가 "좋아", "커밋해줘" 등 최종 승인을 내리면, 작업 내용을 대표하는 의미 있는 한국어 커밋 메시지를 생성하여 Git 커밋을 수행합니다.
- **예시:** `feat: 메인 페이지 컴포넌트 구조 최적화 및 린트 에러 해결`
