# [CHG-260420-04] URL 업데이트 로직 최적화 및 히스토리 관리 개선 (2026-04-20)

## 1. 목적 (Purpose)

- 원두 탐색 페이지에서 필터를 토글하거나 검색어를 입력할 때마다 브라우저 히스토리가 생성되어 '뒤로 가기' 사용성이 저하되는 문제를 해결합니다.
- transient한 상태 변경에는 `router.replace`를 사용하여 히스토리 오염을 방지합니다.

## 2. 중간 과정 및 변경 사항 (Process & Deviations)

- **선택적 업데이트**: `updateUrl` 함수에 `options.replace` 인자를 추가하여 상황에 맞는 내비게이션 방식을 선택할 수 있도록 개선하였습니다.
- **적용 범위**: `handleFilterChange`와 `handleSearchChange`에는 `replace: true`를 적용하고, `handleReset`(초기화)에는 `replace: false`를 적용하여 명시적으로 히스토리를 쌓도록 설정하였습니다.
- **제외 사항**: 사용자 요청에 따라 라이브 검색 및 디바운스 로직은 이번 변경 범위에서 제외되었습니다.

## 3. 결과 (Result)

- `apps/web/app/(main)/beans/page.tsx` 내 네비게이션 로직 최적화 완료.
- 필터 변경 시 브라우저 히스토리가 쌓이지 않으며, 검색 페이지 진입 전 상태로의 복귀가 원활해짐.

## 4. 검증 및 지표 (Verification & Metrics)

- [x] 필터 토글 후 브라우저 뒤로 가기 시 이전 필터 상태가 아닌 '검색 페이지 이전 페이지'로 바로 이동함 확인
- [x] 검색어 입력 후 엔터 입력 시 히스토리가 추가되지 않고 URL만 교체됨 확인
- [x] 초기화 버튼 클릭 시 새로운 히스토리 엔트리가 생성됨 확인
