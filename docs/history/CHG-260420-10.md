# [CHG-260420-10] BeanCard 및 RatingScale 접근성(A11y) 고도화 (2026-04-20)

## 1. 목적 (Purpose)

- `BeanCard`의 정보 표시용 지표(`RatingScale`)가 불필요하게 키보드 포커스를 받는 문제를 해결하고, 호버 시에만 나타나던 상세 정보를 키보드 사용자 및 스크린 리더 사용자도 인지할 수 있도록 접근성을 강화합니다.

## 2. 중간 과정 및 변경 사항 (Process & Deviations)

- **RatingScale 접근성 보강**:
  - `readOnly` 속성을 추가하여, 정보 표시가 목적인 경우 내부 요소를 `<button>`이 아닌 `<motion.div>`로 렌더링하도록 변경했습니다.
  - 이를 통해 키보드 탭(Tab) 이동 시 불필요한 포커스 점유를 방지하고 시각적 이미지(`role="img"`)로 인식되도록 구조를 개선했습니다.
- **BeanCard 키보드 인터랙션 개선**:
  - 프로필 오버레이에 `group-focus-within` 클래스를 적용하여, 카드가 키보드 포커스를 받았을 때도 상세 정보가 시각적으로 노출되도록 수정했습니다.
  - 동시에 기존의 하단 그라데이션 오버레이와 텍스트 영역은 포커스 시 부드럽게 사라지도록 처리하여 가독성을 확보했습니다.
- **시맨틱 마크업 보완**:
  - `Link`와 `Image`에 명확한 `aria-label` 및 상세한 `alt` 텍스트를 추가하여 보조 공학 기기 사용자의 경험을 개선했습니다.

## 3. 결과 (Result)

- `packages/ui-library/components/RatingScale.tsx` 수정 완료.
- `apps/web/components/common/cards/BeanCard.tsx` 접근성 개선 완료.
- 시각적 디자인과 접근성 표준의 조화 달성.

## 4. 검증 및 지표 (Verification & Metrics)

- [x] 키보드 탭 이동 시 더 이상 `RatingScale` 내부 도트에 포커스가 잡히지 않음.
- [x] 카드 포커스 시 검은 오버레이와 상세 정보가 정상적으로 나타남.
- [x] 스크린 리더를 위한 `aria-label` 정상 적용 확인.
