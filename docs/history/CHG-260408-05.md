# [CHG-260408-05] 로그인 페이지 애니메이션 타이밍 최적화 (2026-04-08)

## 1. 목적 (Purpose)

- 사용자 피드백 반영: 로그인 페이지 진입 시 헤더 애니메이션 이후 소셜 로그인 버튼이 나타나는 타이밍이 너무 늦어 UI가 느리게 느껴지는 문제 해결.
- 애니메이션 흐름 개선: 레이아웃 요소 간의 시각적 공백을 제거하고 프리미엄한 반응성 확보.

## 2. 중간 과정 및 변경 사항 (Process & Deviations)

- **코드 분석**: `SocialLoginSection.tsx`의 `delayChildren`이 `0.8s`로 과도하게 높게 설정되어 있음을 확인.
- **타이밍 조정**:
  - `SocialLoginSection.tsx`: `delayChildren`을 `0.8s` -> `0.3s`로 단축.
  - `LoginPage.tsx`: 부모 컨테이너의 `staggerChildren`을 `0.15s` -> **0.05s**로 대폭 축소하여 즉각적인 로딩감 제공.
- **결과**: 헤더 애니메이션 도중에 소셜 로그인 버튼들이 매우 기민하게 뒤따라 나타나도록 수정됨.

## 3. 결과 (Result)

- 소셜 로그인 버튼 등장 시점 약 0.5초 단축.
- 페이지 전체 애니메이션이 500ms 내외로 완료되어 기민한 사용자 경험 제공.

## 4. 검증 및 지표 (Verification & Metrics)

### 테스트 전략 및 항목

- [x] 브라우저 검증 (`http://localhost:3000/login`)
- [x] 페이지 로드 후 200ms~500ms 사이의 요소 렌더링 상태 확인
- [x] 헤더와 버튼 사이의 시각적 공백 제거 여부 확인
- [x] 애니메이션 브라우저 레코딩 및 스크린샷 증빙 (`login_animation_fix_check`)

### 검증 결과

- 200ms 시점부터 헤더와 버튼이 유기적으로 나타나기 시작함.
- 500ms 시점에 모든 인터랙티브 요소가 활성화됨.
