# [CHG-260417-01] Aroma 명칭 통일 및 메인 페이지 향미 섹션 레이아웃 고도화 (2026-04-17)

## 1. 목적 (Purpose)

- 시스템 전반에서 혼용되던 ‘Tasting’ 용어를 ‘Aroma’로 통일하여 도메인 모델의 일관성을 확보합니다.
- 메인 페이지의 향미 섹션을 사용자 친화적으로 개선하고, 원두 탐색 페이지와의 데이터 연동성을 강화합니다.
- 디바이스별로 최적화된 카드 개수를 노출하여 시각적 완성도를 높입니다.

## 2. 중간 과정 및 변경 사항 (Process & Deviations)

- **데이터 레이어 통합**: `beans.ts`에 `AROMA_DEFINITIONS`를 구축하여 국문/영문 명칭과 이미지 URL을 중앙 관리하도록 리팩토링했습니다. 이를 통해 `main.ts`가 별도의 데이터를 가지지 않고 중앙 데이터를 참조하게 되어 유지보수성을 높였습니다.
- **용어 일원화**: `FlavorNotes`, `FlavorCard` 등의 컴포넌트를 각각 `AromaNotes`, `AromaCard`로 변경하고 내부 Props 명칭을 표준화했습니다.
- **필터링 연동**: 아로마 카드 클릭 시 영문 ID를 포함한 URL 파라미터(`?aromaId=`)를 생성하고, 원두 탐색 페이지(`BeansPage`)에서 이를 감지하여 초기 필터 상태로 자동 반영하도록 구현했습니다. `useSearchParams` 사용에 따른 `Suspense` 래퍼를 추가했습니다.
- **반응형 레이아웃**: 9개의 아로마 카드가 홀수개로 배치되어 불균형해 보이는 문제를 해결하기 위해, CSS를 활용하여 데스크톱 8개, 태블릿 6개, 모바일 4개(모두 2줄 구성)로 노출 개수를 제한했습니다.

## 3. 결과 (Result)

- 메인 페이지 향미 섹션에 "전체 보기" 버튼이 추가되었으며, 디바이스별로 4/6/8개의 카드가 레이아웃 흐름에 맞게 노출됩니다.
- 아로마 카드를 클릭하면 해당하는 영문 ID로 필터링된 원두 리스트를 즉시 확인할 수 있습니다.
- 코드 베이스 내 `Tasting` 용어가 `Aroma`로 통일되었습니다.

## 4. 검증 및 지표 (Verification & Metrics)

### 테스트 전략 및 항목

- **Visual / Responsive Verification**: 브라우저 서브에이전트를 통해 데스크톱(8개), 태블릿(6개), 모바일(4개) 노출 상태를 검증했습니다.
- **Navigation Test**: 메인 페이지 카드 클릭 시 `/beans?aromaId=caramel` 등으로의 이동 및 필터링 자동 적용 여부를 확인했습니다.
