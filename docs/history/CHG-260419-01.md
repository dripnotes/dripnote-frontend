# [CHG-260419-01] BeanCard 디자인 통합 및 호버 프로필 오버레이 구현 (2026-04-19)

## 1. 목적 (Purpose)

- 메인 페이지의 `RecommendedBeanCard`와 원두 목록 페이지의 `BeanCard` 디자인이 상이하여 발생하는 시각적 불일치를 해결합니다.
- 중복된 카드 컴포넌트를 제거하여 유지보수 효율을 높입니다.
- `BeanCard`에 호버 시 원두의 상세 프로필(산미, 단맛, 쓴맛, 바디감, 로스팅 등)을 보여주는 오버레이를 추가하여 사용자 경험(UX)을 고도화합니다.

## 2. 중간 과정 및 변경 사항 (Process & Deviations)

- **로스팅 데이터 구조 혁신**: 로스팅 정보를 기존 텍스트 레이블에서 1-3단계 숫자형(`RoastingType`)으로 전환하여 바디감 지표와 일관성을 확보하였습니다.
- **필터 UI 고도화**: `BeanFilterPanel` 및 `BeanFilterDrawer`에서 로스팅 선택 방식을 칩(Chip) 기반 다중 선택에서 `RatingBar` 기반 단일 선택(1: Light, 2: Medium, 3: Dark)으로 변경하여 직관성을 높였습니다.
- **BeanCard 시각적 개선**: 
  - 호버 오버레이 내 'Roasting' 지표(기존 Roast)를 목록 최하단(`Body` 아래)으로 재배치하여 정보의 위계를 최적화하였습니다.
  - 모든 지표의 도트에 `BeanFilterPanel.tsx`와 동일한 단계별 농도 차(Amber scale: #fde68a ~ #d97706)를 적용하여 시각적 전달력을 극대화하였습니다.
  - 기존 로스팅 배지 위치에 임의의 로스터리 마크(Coffee 아이콘)를 추가하여 향후 실제 브랜드 로고 적용을 위한 공간을 확보하였습니다.

## 3. 결과 (Result)

- `RoastingType` 숫자형 전환 및 관련 필터링 로직 업데이트 완료.
- 로스팅 필터 UI가 영문 가이드를 포함한 `RatingBar`로 교체 완료.
- `BeanCard` 디자인이 로스터리 중심의 프리미엄 레이아웃으로 업데이트 완료.

## 4. 검증 및 지표 (Verification & Metrics)

### 테스트 전략 및 항목

- **Filter Accuracy**: 로스팅 단계를 조절할 때 URL 파리미터(`?roasting=n`)가 올바르게 전송되며 필터링 결과가 정확한지 확인.
- **Visual Consistency**: 카드 호버 시 로스트와 바디감이 동일한 3단계 도트 스타일로 표현되는지 확인.
- [x] 로스팅 지표 숫자형(1/2/3) 전환 및 필터링 확인
- [x] 로스팅 필터 영문 레이블(Light/Medium/Dark) 적용 확인
- [x] 카드 내 로스터리 마크 플레이스홀더 추가 확인
