# CHG-260416-01: 원두 탐색 검색창(BeanSearchBar) 통합 및 UI 구조 일원화

## 1. 개요 (Context)

- **목적**: 띄엄띄엄 분리되어 있던 검색창 영역을 필터 구역과 합쳐서 탐색 집중도와 일관성을 높입니다.
- **배경**: 원두 탐색(검색+필터) 과정에서 사용자의 시선이 분산되는 것을 막고, 모바일과 데스크톱 양쪽 환경에서 모두 더 효율적으로 공간을 활용하기 위해 통합이 필요했습니다.

## 2. 작업 내용 (Action)

1. **`BeanSearchBar` 수정**:
   - `max-w-xs` 등 고정된 최대 너비를 제거하고 부모 컨테이너 크기에 맞춰 100%(`w-full`) 채우도록 변경.
   - Placeholder 텍스트를 좀 더 통합적인 역할에 맞게 "검색어를 입력하세요"로 변경.
2. **`BeanFilterPanel` 통합 (Desktop)**:
   - 검색어 상태(`searchQuery`, `onSearchChange`)를 Props로 주입.
   - 패널 내 최상단 위치(Aroma 위)에 `BeanSearchBar` 추가.
3. **`BeanFilterDrawer` 통합 (Mobile)**:
   - 모바일 하단 드로어의 최상단에 동일하게 검색창 추가 및 상태 연동.
4. **`BeansPage` UI 클린업**:
   - 상단에서 겉돌던 독립적인 검색창을 제거하고, 필터와 카드 목록 사이에 결과 개수를 명확히 보여주는 요약 헤더 구역으로 깔끔하게 정리.
   - 상단의 헤더(`Header.tsx`)와의 간격 확보를 위해 패딩(`pt-16`) 적용.
5. **명세서 동기화**:
   - `specs/beans-page.md`를 변경된 내용(레이아웃, 아키텍처 트리, Props 구조 등)에 맞게 갱신.

## 3. 결과 (Result)

- 검색과 필터 관련 시퀀스가 하나의 컴포넌트로 일원화되어 조작의 흐름이 단방향으로 통합되었습니다.
- 모바일 환경에서도 필터 드로어를 통해 한 곳에서 조건 설정 및 검색이 가능해졌습니다.
- 관련 명세 문서를 모두 실제 구현과 일치시켰습니다.
