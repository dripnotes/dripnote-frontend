# 변경 이력 (Change History) - CHG-260408-09

## 1. 목적 (Purpose)

- `Button` 컴포넌트의 스타일 정합성 확보: 기본 `buttonVariants` 베이스 클래스에 정의된 `active:scale-[0.98]`과 `icon` 사이즈 변수의 `active:scale-90`이 중복/충돌하여 발생할 수 있는 의도치 않은 애니메이션 동작을 방지합니다.
- 스타일 정의의 명확화: 각 사이즈별로 독립적인 `active` 효과를 정의하여 테일윈드 클래스 병합 과정에서의 불확실성을 제거합니다.

## 2. 과정 및 변경 사항 (Process & Deviations)

- **코드 분석**: `cva`로 정의된 `buttonVariants`의 베이스 테두리 문자열에 `active:scale-[0.98]`이 고정되어 있어, `size="icon"` 적용 시 두 종류의 `active:scale` 클래스가 동시에 존재하게 됨을 확인.
- **스타일 리팩토링**:
  - `packages/ui-library/components/Button.tsx`:
    - 베이스 클래스(string)에서 `active:scale-[0.98]` 속성을 제거.
    - `size` variant의 각 세부 항목(`default`, `sm`, `lg`, `xl`)으로 해당 속성을 이동시켜 명시적으로 정의.
    - `icon` 사이즈는 기존의 `active:scale-90`만 유지하여 베이스 클래스와의 원천적인 충돌 가능성을 차단.

## 3. 결과 (Result)

- `icon` 타입 버튼 클릭 시 90% 축소 애니메이션이 명확하게 적용됨.
- 일반 타입 버튼 클릭 시 기존의 미세한 축소(0.98) 효과가 일관되게 적용됨.
- 컴포넌트 스타일의 예측 가능성 및 견고함 향상.

## 4. 검증 및 지표 (Verification & Metrics)

- **타입 체크**: `pnpm --filter @coffee-service/ui-library type-check`를 수행하여 빌드 이슈 없음을 확인.
- **시각적 검증**:
  - 헤더 아이콘 버튼과 메인 섹션 일반 버튼의 클릭 피드백이 서로 독립적으로 올바르게 작동함을 확인.
  - `twMerge`를 통한 클래스 병합 과정에서 중복된 `active:scale` 클래스가 나타나지 않음을 확인.
