# CHG-260416-03: 원두 페이지 UX 고도화 및 레이아웃 안정화

## 1. 개요 및 목적

- **목표**: 원두 정보 페이지(/beans)의 시각적 가시성을 높이고, 필터링 시 발생하는 레이아웃 흔들림(Layout Shift)을 해결하며, 애니메이션 방식을 사용자 친화적인 페이드(Fade) 효과로 개선합니다.
- **배경**: 저대비 환경에서 필터 요소의 구분이 어렵다는 피드백과, 카드 개수가 변할 때마다 스크롤바 유무에 의해 전체 레이아웃이 좌우로 튀는 현상을 해결하기 위함입니다.

## 2. 상세 작업 내용

### 가시성 및 인터랙션 개선

- `BeanFilterPanel.tsx` 및 `BeanFilterDrawer.tsx` 내 Chip, Rating Bar 요소에 `border` 추가 및 대비 강화.
- 비활성 상태의 텍스트 및 배경 색상 대비를 상향 조정하여 가독성 확보.
- `BeanSearchBar.tsx`의 배경색을 흰색으로 변경하고 `ring` 및 `shadow`를 적용하여 검색 입력 영역 명확화.

### 레이아웃 안정화 (Pixel-perfect Consistency)

- `globals.css`에 `scrollbar-gutter: stable`을 적용하여 스크롤바 생성/소멸 시 페이지 중앙 정렬 기준점이 흔들리는 문제를 근본적으로 해결.
- `page.tsx` 및 `BeanCardList.tsx`의 주요 컨테이너에 `w-full` 및 `items-start`를 적용하여 뷰포트별 일관된 너비 확보.
- `BeanCardList.tsx`의 빈 결과 상태(Empty State) 레이아웃 붕괴 방지를 위해 `w-full` 적용.

### 애니메이션 고도화

- `BeanCard.tsx`에서 부자연스러운 슬라이딩 효과(`layout` 속성) 제거.
- `AnimatePresence`를 활용해 필터 변경 시 모든 카드가 제자리에서 나타나고 사라지는 페이드(Fade) + 미세 스케일(Scale) 효과 적용.
- 필터 변경 시 리스트 전체가 다시 페이드인되도록 컨테이너 `key` 속성 최적화.

## 3. 최종 결과 및 검증

- **레이아웃**: 카드 개수(1개~12개)에 관계없이 우측 섹션의 시작 좌표(X)가 1px의 오차 없이 고정됨을 브라우저 측정을 통해 확인.
- **애니메이션**: 슬라이딩 없이 깔끔한 페이드 효과가 매끄럽게 동작함.
- **문서화**: `specs/beans-page.md`에 모든 변경 사항 반영 완료.
