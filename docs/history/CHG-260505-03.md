# CHG-260505-03: AI 리포트 피드백 적용 및 안정성 강화

## 1. 목적 (Purpose)

- 외부 AI 검토 리포트의 피드백을 수용하여 코드베이스 및 명세서의 정합성을 확보합니다.
- 예외 상황(Network Error, JSON 파싱 에러)에 대한 방어 로직을 보강하여 개발 인프라의 안정성을 높입니다.

## 2. 과정 및 변경 사항 (Process & Deviations)

### 2.1 API Playground 로직 및 UI 개선 (`page.tsx`)

- **응답 파싱 안정화**: `fetch` 이후 `Content-Type`을 검증하여, `application/json`이 아닐 경우 `res.text()`를 통해 텍스트 본문과 HTTP 상태 코드가 포함된 명시적 에러를 던지도록 보완했습니다. (HTML 응답 시 발생하는 SyntaxError 방지)
- **성능 상태 뱃지 보정**: `response.status === 200`을 `response.status >= 200 && response.status < 300`으로 수정하여 201, 204 등의 2xx 응답도 정상으로 표시하도록 개선했습니다.
- **클래스 충돌 제거**: Network Error 타이틀의 `text-white`와 `text-red-500` 충돌을 방지하기 위해 불필요한 `text-white`를 삭제했습니다.

### 2.2 Storage Key 상수화 (`use-auth.ts`, `auth-utils.ts`)

- `auth-utils.ts`에서 사용하던 `baristation-auth-token` 키를 `AUTH_TOKEN_KEY`로 상수화하여 export 하도록 수정했습니다.
- `use-auth.ts`의 크로스탭 이벤트 리스너에서 해당 상수를 참조하도록 하여 스토리지 키 관리의 일원화를 달성했습니다.

### 2.3 문서 및 명세 정합성 확보 (`specs/main-page.md`, `vibe-coding-cycle.md`)

- **브랜드 동기화**: `vibe-coding-cycle.md` 내에 남아있던 Dripnote 명칭을 Baristation으로 교체했습니다.
- **AromaCard 경로**: 스펙의 경로를 실제 컴포넌트 경로(`common/cards/...`)로 수정했습니다.
- **JSON 타입 보정**: `recommendedBeans.id`의 `"1"`을 숫자 `1`로 수정했습니다.
- **카피라이트 연도**: `© 2026`을 동적 연도 템플릿 `{currentYear}`로 변경하고 연산 방법을 명시했습니다.

## 3. 결과 (Result)

- 일관된 Storage Key 관리로 인증 토큰 동기화의 안정성이 보장되었습니다.
- API Playground가 비정상적인 서버 응답에서도 크래시 없이 명확한 에러 메시지를 제공합니다.
- 명세서와 실제 구현체 간의 불일치가 모두 해소되어, 에이전트의 문서 참조 신뢰도가 향상되었습니다.

## 4. 검증 및 지표 (Verification & Metrics)

- 정적 분석 및 ESLint 검증 통과 (TypeScript 문법 오류 없음)
- Playground 화면의 Network Error 시 렌더링 정상 동작 확인
