## 2. 상세 내역 (Detailed Changes)

### 2.1 검색 정책 변경 (Enterprise Search)

- **대상**: `BeanSearchBar.tsx`
- **변경 사항**:
  - 기존 300ms Debounce 기반의 실시간 검색(Live Search)을 폐기.
  - 입력 후 `Enter` 키를 눌렀을 때만 검색이 실행되도록 이벤트 연동.
  - 불필요해진 `useRef` 및 타이머 로직 제거로 코드 단순화.

### 2.2 지연 필터링(Deferred Filtering) 도입

- **대상**: `BeanFilterPanel.tsx` (Desktop), `BeanFilterDrawer.tsx` (Mobile)
- **변경 사항**:
  - 컴포넌트 내부 로컬 상태(`localFilters`) 도입.
  - 필터(Aroma, Flavor, Body, Roasting) 조작 시 즉시 반영되지 않고 로컬 상태만 갱신.
  - "적용하기" 버튼 클릭 시에만 부모 상태(`onChange`)로 전달되어 결과 목록 갱신 및 드로어 닫힘 수행.

### 2.3 스티키(Sticky) UI 적용

- **대상**: `BeanFilterPanel.tsx`, `BeanFilterDrawer.tsx`
- **변경 사항**:
  - 패널 및 드로어 최하단에 `sticky bottom-0`을 적용한 "적용하기" 버튼 배치.
  - 스크롤의 위치와 관계없이 항상 버튼이 화면 하단에 고정되어 다중 필터 선택 후 제출이 용이하도록 개선.
  - (모바일) 드로어 내 불필요한 상단 닫기(X) 버튼을 제거하고 "적용하기" 버튼으로 인터랙션 단일화.

## 3. 검증 결과 (Verification)

- **검색창**: 타이핑 시 리스트 변화 없음 → 엔터 입력 시 필터링 수행 확인.
- **필터 패널**: 다중 필터 클릭 시 리스트 변화 없음 → '적용하기' 클릭 시 애니메이션과 함께 반영 확인.
- **스티키 버튼**: 긴 필터 영역에서도 버튼이 하단에 항상 고정되어 노출됨을 확인.
