# CHG-260416-02: 레이아웃 아키텍처 개편 및 모바일 앱 뷰(Bottom Nav) 도입

## 1. 개요 (Context)

- **목적**: 모바일 뷰에서의 사용성을 극대화하기 위해 헤더를 하단 탭 바(Bottom Navigation Bar) 형식으로 변경하고, 각 페이지에 분산되어 있던 여백 패딩 로직을 단일 선언형 컨테이너(`PageContainer`)로 통합 관리합니다.
- **배경**: 기존 다크 테마의 상단 고정 헤더는 모바일 화면에서 한정된 공간을 차지하고 한 손 접근성이 떨어졌습니다. 또한 각 페이지(`page.tsx`, `beans/page.tsx` 등)마다 모바일/데스크톱 상황에 맞춰 임시방편으로 패딩(`pt-16`, `pb-16`)을 다르게 넣는 등 보일러플레이트 중복이 발생하고 있었습니다.

## 2. 작업 내용 (Action)

1. **`Header.tsx` 모바일 하단 탭 바(Bottom Nav) 전환**:
   - 모바일 해상도(`< 768px`)에서는 헤더를 하단(`bottom-0`)으로 고정.
   - 밝은 테마(`bg-white`)로 전환되며 로고를 숨기고, 5개의 네비게이션 아이콘(Home, Beans, Classes, Saved, My)을 탭 바 형식으로 균등 배치.
   - 데스크톱 해상도(`md:top-0`)에서는 기존 상단 다크/투명 테마 구조 유지.
2. **선언형 `PageContainer` 컴포넌트 신설 (`components/layout/PageContainer.tsx`)**:
   - 최상단 `layout.tsx`와 개별 `page.tsx` 컨텐츠 사이에 위치하는 여백 관리용 공용 컨테이너 작성.
   - `withHeaderOffset` 속성으로 상단 오버랩/논-오버랩 상태를 선언적으로 관리.
3. **각 페이지 보일러플레이트 패딩 제거**:
   - `beans/page.tsx` 및 `(main)/page.tsx`의 하드코딩된 Tailwind 여백 속성을 지우고 `<PageContainer>`로 마이그레이션.
4. **`Footer.tsx` 하단 여백 추가**:
   - 화면 끝까지 스크롤 시 푸터 내용이 모바일 하단 탭 바에 가려지지 않도록 `pb-28` 여백 확보.
5. **AI 전역 개발 원칙(`.agents/rules/global.md`) 갱신**:
   - 페이지 레이아웃 래퍼 강제 적용.
   - Zustand를 활용한 전역 오버레이 상태 중앙 제어.
   - Server Action과 TanStack Query 하이브리드 혼용 체계 기술.
   - Compound Component 설계 패턴 도입 가이드 추가.

## 3. 결과 (Result)

- 모바일 사용자 경험(UX)이 네이티브 앱과 같은 Bottom Tab Bar 형태로 대폭 개선되었습니다.
- 페이지별로 분산된 레이아웃 여백 관리가 `PageContainer`로 완벽히 추상화되어 향후 화면 개발 시 확장성과 생산성이 크게 상승했습니다.
- AI 어시스턴트 코드 기반의 퀄리티 보장을 위한 견고한 아키텍처 룰 셋(Compound Pattern 등)이 문서화되었습니다.
