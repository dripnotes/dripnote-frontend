# [CHG-260417-03] VisualCard 공통 컴포넌트화 및 카드 UI 리팩토링 (2026-04-17)

## 1. 목적 (Purpose)

- `AromaCard`, `BeanCard`, `RecommendedBeans` 등 여러 섹션에서 중복되던 카드 UI 로직을 단일화합니다.
- 디자인 시스템 레벨의 `VisualCard` (Compound Component)를 구축하여 시각적 품질과 유지보수성을 향상시킵니다.

## 2. 중간 과정 및 변경 사항 (Process & Deviations)

- **UI Library 통합**: `packages/ui-library`에 `VisualCard`를 추가하여 모든 앱에서 동일한 디자인 토큰과 애니메이션 라이브러리를 공유하게 했습니다.
- **슬롯(Slot) 패턴 적용**: `radix-ui/react-slot`을 사용하여 `Link` 컴포넌트와 유연하게 결합할 수 있도록 설계했습니다.
- **프리셋 레이어 분리**: `apps/web/components/common/cards/`에 각 도메인별 전용 프리셋을 두어 컨테이너 컴포넌트의 복잡도를 획기적으로 낮췄습니다.
- **모션 통합**: 각기 다른 파일에 흩어져 있던 `framer-motion` 설정들을 `VisualCard` 내부 프로퍼티로 추상화했습니다.

## 3. 결과 (Result)

- `ui-library`에 강력하고 유연한 `VisualCard` 시스템 구축 완료.
- 메인 페이지 및 원두 탐색 페이지의 모든 카드 UI를 새로운 시스템으로 100% 전환 완료.
- 불필요한 중복 코드 제거 및 섹션 컴포넌트의 가독성 향상.

## 4. 검증 및 지표 (Verification & Metrics)

### 테스트 전략 및 항목

- **Visual Regression**: 브라우저 서브에이전트를 통한 디자인 일관성 검증 완료.
- **Interaction Test**: 호버 스케일 및 이동 애니메이션 동작 확인.
- **Respionsive Test**: 모바일/데스크톱 환경에서의 카드 레이아웃 정합성 확인.
