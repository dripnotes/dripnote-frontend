# 작업 이력 (CHG-260421-01)

## 1. 개요 (Overview)

Next.js 15의 파괴적 변경 사항인 `params`의 Promise 전환에 대응하고, `VisualCard` 컴포넌트 내의 비표준 타입 캐스팅(`as any`)을 제거하여 전반적인 프로젝트의 타입 안정성과 프레임워크 호환성을 확보합니다.

## 2. 변경 내용 (Changes)

### 2.1 Next.js 15 `params` Promise 대응

Next.js 15에서는 서버 컴포넌트의 `params`와 `searchParams`가 동기 객체가 아닌 Promise로 제공됩니다. 이를 반영하기 위해 원두 상세 페이지를 업데이트했습니다.

- **[MODIFY]** [apps/web/app/(main)/beans/[id]/page.tsx](<apps/web/app/(main)/beans/[id]/page.tsx>)
  - `Props` 인터페이스에서 `params`를 `Promise<{ id: string }>`으로 변경.
  - `generateMetadata` 및 `BeanDetailPage` 컴포넌트에서 `await params`를 수행한 후 속성에 접근하도록 수정.
  - 페이지 컴포넌트를 `async` 함수로 전환.

### 2.2 `VisualCard` 타입 안정성 개선

`VisualCard.Image` 컴포넌트에서 런타임 오류 방지 및 타입 검사를 강화하기 위해 리팩토링을 진행했습니다.

- **[MODIFY]** [packages/ui-library/components/VisualCard.tsx](packages/ui-library/components/VisualCard.tsx)
  - `VisualImage` 컴포넌트에서 `hoverScale`, `asChild` 등의 커스텀 props를 구조 분해 할당으로 명시적으로 분리.
  - 나머지 `motion.img` 관련 props만 하위 요소로 전달하도록 수정하여 `{...(props as any)}`와 같은 위험한 타입 단언을 제거.

## 3. 검증 결과 (Verification)

### 3.1 빌드 및 타입 체크

- `pnpm --filter @coffee-service/web build` 실행 결과 성공 (Exit Code 0). Next.js 15 빌드 타임의 `params` 타입 체크를 통과함.
- `pnpm --filter @coffee-service/ui-library run type-check` 실행 결과 성공 (Exit Code 0). 컴포넌트 라이브러리의 타입 정합성 확인.

### 3.2 수동 확인

- 원두 상세 페이지로의 라우팅 및 데이터 로드가 정상적으로 동작함을 확인.
- 원두 카드 이미지의 호버 스케일 애니메이션이 기존과 동일하게 부드럽게 작동함을 확인.

## 4. 향후 과제 (Next Steps)

- 향후 추가되는 모든 동적 루트(`[slug]`, `[id]` 등)에서 `params`를 Promise로 처리하는 패턴을 기본으로 적용.
- `ui-library` 내 다른 컴포넌트들에서도 유사한 `as any` 패턴이 발견될 경우 정기적인 리팩토링 수행.
