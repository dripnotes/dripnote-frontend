# 작업 이력 (CHG-260421-02)

## 1. 개요 (Overview)

컴포넌트의 접근성 및 네비게이션 로직을 정교화하고, 시각적 일관성과 문서 정합성을 확보하기 위한 폴리싱(Polishing) 작업을 수행했습니다. 특히 보조 공학 기기 지원을 위한 속성 추가와 프레임워크 최적화를 중점적으로 다루었습니다.

## 2. 변경 내용 (Changes)

### 2.1 컴포넌트 접근성 및 네비게이션 개선

- **[MODIFY]** [apps/web/app/(main)/beans/[id]/\_components/BeanDetailHero.tsx](<apps/web/app/(main)/beans/[id]/_components/BeanDetailHero.tsx>)
  - 북마크 토글 버튼에 `aria-pressed` 속성을 추가하여 현재 토글 상태를 스크린 리더 등 보조 공학 기기에 알림.
  - 뒤로 가기 기능을 `router.back()`에서 확정적인 Next.js `<Link href="/beans">`로 교체하여 페이지 전환의 예측 가능성 확보.
  - `AromaType`을 `import type`으로 분리하여 런타임 최적화 및 미사용 `useRouter` 임포트 제거.

### 2.2 인터랙션 및 애니메이션 최적화

- **[FIX]** [packages/ui-library/components/RatingScale.tsx](packages/ui-library/components/RatingScale.tsx)
  - `initial={isActive ? false : { ... }}` 로직을 적용하여, 페이지 로드 시 기존에 선택된 레이팅 값이 시각적으로 번쩍이며 애니메이션되는 현상을 방지.
- **[REFACTOR]** [packages/ui-library/components/VisualCard.tsx](packages/ui-library/components/VisualCard.tsx)
  - Framer Motion의 `whileHover`에 의존하던 이미지 스케일링 로직을 Tailwind CSS의 `group-hover:scale-110` 방식으로 전환.
  - 성능 최적화 및 카드 전체 영역 호버 시 보다 즉각적인 시각적 피드백 제공.

### 2.3 문서 및 명세서 동합성 확보

- **[FIX]** [docs/history/CHG-260421-01.md](docs/history/CHG-260421-01.md)
  - 로컬 파일 시스템 경로(`file:///...`)를 저장소 상대 경로로 변환하여 이식성 문제 해결.
- **[SYNC]** [specs/beans-page.md](specs/beans-page.md)
  - 실제 구현된 지표 순서(산미·단맛·밸런스)에 맞춰 명세 텍스트 수정.
  - `BeanInfo` 및 `BeanCardProps`의 내비게이션 기준을 `id`가 아닌 `link` 프로퍼티로 표준화.
- **[SYNC]** [specs/main-page.md](specs/main-page.md)
  - `BeanItem` 인터페이스에서 낡은 속성명인 `bitterness`를 `balance`로 교체.
  - `Success JSON` 예시 데이터를 최신 응답 규격(`name`, `aromaImageUrl` 등)에 맞게 업데이트.

## 3. 검증 결과 (Verification)

### 3.1 빌드 및 품질 체크

- `pnpm run type-check` 실행을 통해 모든 컴포넌트의 타입 안정성 확인.
- `next lint` 실행을 통해 접근성 속성 누락 및 미사용 임포트 부재 확인.

### 3.2 수동 확인

- 원두 상세 페이지의 "Back" 버튼이 원두 리스트 페이지(`/beans`)로 정확히 이동함을 확인.
- 카드 호버 시 이미지가 부드럽게 확대되며, 레이팅 컴포넌트 초기 로드 시 시각적 잡음이 없음을 확인.
