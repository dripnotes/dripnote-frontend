# [CHG-260408-07] 로그인 페이지 모바일 뷰 최적화 및 스크롤 제거 (2026-04-08)

## 1. 목적 (Purpose)

모바일 브라우저에서 헤더와 버튼 사이의 과도한 여백으로 인해 페이지가 100vh를 초과하여 스크롤이 발생하는 문제를 해결합니다. 모든 요소를 한 화면 내에 배치하여 사용자 경험을 고도화합니다.

## 2. 중간 과정 및 변경 사항 (Process & Deviations)

- **LoginPage.tsx 수정**:
  - `min-h-screen`을 `h-[100dvh]`로 변경하여 유동적인 모바일 뷰포트에 대응.
  - `overflow-hidden` 적용으로 스크롤 원천 차단.
  - 푸터(`footer`)의 위치를 모바일 환경에 맞춰 상단으로 소폭 조정(`bottom-6`).
- **LoginHeader.tsx 수정**:
  - 상하단 패딩 축소 (`pt-12 pb-6`) 및 폰트 크기 미세 조정(`text-5xl`).
- **SocialLoginSection.tsx 수정**:
  - 소셜 버튼 간의 간격을 `20px`에서 `12px`로 축소하여 수직 공간 확보.
  - 구분선(`Divider`) 및 하단 안내 문구의 여백 최적화.

## 3. 결과 (Result)

- 아이폰 SE 등 작은 화면에서도 스크롤 없이 모든 기능을 한눈에 확인 가능.
- 사용자의 피드백을 반영한 기민한 UX 구현 완료.
