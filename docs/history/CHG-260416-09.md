# [CHG-260416-09] 필터 초기화(Reset) 시 로컬 상태 동기화 누락 수정

## 1. 개요 (Overview)

사용자가 필터를 변경한 후 '적용하기'를 누르지 않은 상태(dirty draft)에서 '초기화' 버튼을 클릭할 때, UI에 표시되는 로컬 상태가 즉시 초기화되지 않는 문제를 해결했습니다.

## 2. 상세 변경 내역 (Detailed Changes)

### 2.1 필터 패널 및 드로어 상태 동기화 로직 개선

- **문제 원인**: 기존 `useEffect`는 `filters` prop의 참조값 변화에만 의존했습니다. 만약 부모 상태가 이미 기본값(`DEFAULT_FILTERS`)인 상태에서 '적용하기'를 누르지 않고 '초기화'를 누르면, 부모 상태는 변하지 않아 자식의 `useEffect`가 실행되지 않고 로컬 상태가 그대로 남는 현상이 발생했습니다.
- **해결 방법**: `onReset` 콜백 호출 시 `setLocalFilters(DEFAULT_FILTERS)`를 명시적으로 호출하여 부모 상태 변화와 관계없이 로컬 UI를 즉시 초기화하도록 수정했습니다.
- **적용 대상**: `BeanFilterPanel.tsx`, `BeanFilterDrawer.tsx`

### 2.2 공통 필터 상수 활용

- `lib/api/beans.ts`에서 정의된 `DEFAULT_FILTERS` 상수를 임포트하여 초기화 값의 정합성을 보장했습니다.

## 3. 검증 결과 (Verification)

- [x] 데스크톱 환경(Panel)에서 필터 선택 후 '적용하기' 전 '초기화' 클릭 시 칩들이 즉시 회색으로 돌아오는 것 확인.
- [x] 모바일 드로어(Drawer)에서 동일 시나리오 테스트 완료.
- [x] '적용하기'를 누른 후의 전역 필터 초기화 동작과도 상충되지 않음을 확인.
