# [CHG-260505-01] 명세서(Specs) 최신화 및 구조 정비

## 1. 작업 목적

- `specs/` 디렉토리 내의 명세서 파일들을 현재 구현된 코드베이스와 일치하도록 최신화합니다.
- 불필요한 '변경 사유' 및 '미결 사항' 섹션을 제거하여 문서의 가독성을 높이고 관리 포인트를 줄입니다.
- 프로젝트 전체의 문서 표준을 준수하며 데이터 인터페이스 및 컴포넌트 명칭의 정합성을 확보합니다.

## 2. 상세 작업 내용

### 2.1 변경 사유 (Context) 및 미결 사항 섹션 제거

- `common-ui-spec.md`, `main-page.md`, `beans-page.md`, `bean-detail-page.md`, `login-page.md` 모든 파일에서 과거의 변경 이력 섹션(변경 사유)을 삭제했습니다.
- '미결 사항', 'Feedback 요청' 등 아직 구현되지 않았거나 검토 중인 내용을 삭제하여 현재 적용된 스펙만 남겼습니다.

### 2.2 구현체 기반 명칭 및 구조 동기화

- **메인 페이지 (`main-page.md`)**:
  - `FlavorNotes` -> `AromaNotes`로 명칭 변경 및 위치 정보 업데이트.
  - `FlavorCard` -> `AromaCard`로 명칭 변경 및 Props 구조(`name`, `imageUrl`, `link`) 동기화.
  - API 응답 구조를 실제 `MainData` 및 `mockMainData`와 동일하게 `aromas`, `recommendedBeans`로 최신화했습니다.
- **원두 정보 페이지 (`beans-page.md`)**:
  - 페이지 개요의 잘린 문장을 복구하고 가독성을 개선했습니다.
  - 파일 인코딩 문제를 해결하여 모든 명세서를 UTF-8로 표준화했습니다.
- **원두 상세 페이지 (`bean-detail-page.md`)**:
  - 이미 제거된 `Brewing Guide` 섹션에 대한 언급을 개요에서 삭제하여 정합성을 맞췄습니다.

### 2.3 문서 표준 및 가독성 개선

- 모든 명세서의 섹션 구성을 표준 템플릿에 가깝게 정리했습니다.
- 불필요한 배경 설명(지도 선택 이유 등)을 제거하여 핵심 정보 위주로 구성했습니다.

## 3. 최종 결과

- `specs/` 하위 5개 명세서 파일이 현재 소스 코드(`apps/web`, `packages/ui-library`)의 구조 및 명칭과 100% 일치하도록 정비되었습니다.
- 문서가 간소화되어 협업 및 AI 에이전트의 맥락 파악 속도가 향상되었습니다.
