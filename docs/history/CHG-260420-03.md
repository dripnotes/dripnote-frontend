# [CHG-260420-03] VisualCard asChild 패턴 개선 및 카드 컴포넌트 표준화 (2026-04-20)

## 1. 목적 (Purpose)

- `VisualCard.Root`가 `asChild` 패턴을 사용할 때 발생하는 불필요한 중첩 `div` 문제를 해결하여 마크업 구조를 최적화합니다.
- `AromaCard` 및 `BeanCard`의 루트 요소를 `Link`로 변경하여 웹 표준 및 SEO 성능을 개선합니다.
- `BeanCard` 내에 개별 구현되어 있던 지표(Flavor Profile)를 `RatingScale` 공통 컴포넌트로 통합하여 디자인 일관성을 확보합니다.

## 2. 중간 과정 및 변경 사항 (Process & Deviations)

- **UI 라이브러리 최적화**: `VisualCard.Root` 내부에서 `motion(Slot)`을 사용하도록 수정하여, `asChild` 속성 활성 시 모션 프로퍼티가 자식 요소(Link 등)에 직접 주입되도록 개선하였습니다.
- **구조적 간소화**: `AromaCard`와 `BeanCard`에서 `Link`를 감싸던 불필요한 `div` 클래스들을 제거하고, `VisualCard.Root`가 직접 `Link`의 역할을 수행하도록 리팩토링하였습니다.
- **지표 컴포넌트 통합**: `BeanCard`의 `ProfileIndicator` 내부 로직을 `RatingScale(variant="sm")`으로 교체하여 앱 전반의 지표 디자인을 통일하였습니다.

## 3. 결과 (Result)

- `packages/ui-library/components/VisualCard.tsx` 최적화 완료.
- `apps/web/components/common/cards/AromaCard.tsx` 및 `BeanCard.tsx` 리팩토링 완료.
- 카드 컴포넌트의 DOM 중첩 깊이 축소 및 지표 디자인 표준화 완료.

## 4. 검증 및 지표 (Verification & Metrics)

- [x] 브라우저 개발자 도구를 통한 `VisualCard.Root` 중첩 `div` 제거 확인
- [x] `AromaCard`, `BeanCard` 클릭 시 페이지 이동 정상 동작 확인
- [x] `BeanCard` 호버 오버레이 내 지표가 `RatingScale` 스타일로 정상 렌더링됨을 확인
