# [CHG-260408-06] Button 컴포넌트 icon variant 추가 및 Header 리팩토링 (2026-04-08)

## 1. 목적 (Purpose)

- UI 라이브러리 고도화: 북마크, 로그인 등 아이콘 형태의 버튼을 위한 전용 `icon` variant와 `size`를 추가하여 재사용성을 높임.
- 인터랙션 개선: 호버 시 커서 변경(`pointer`), 배경색 변화, 크기 확대(`scale-110`) 등을 통해 더 명확하고 프리미엄한 피드백 제공.
- 코드 정합성 확보: `Header.tsx` 내의 수동 구현된 버튼들을 라이브러리 컴포넌트로 대체하여 중복 스타일 제거 및 유지보수성 향상.

## 2. 중간 과정 및 변경 사항 (Process & Deviations)

- **Button 컴포넌트 수정**:
  - **API 정교화**: `variant: "icon"`을 제거하고, 대신 `size: "icon"`에 원형 레이아웃(`rounded-full`) 및 호버 애니메이션(`hover:scale-110`)을 통합하여 역할 분리를 명확히 함.
  - **다형성 지원**: `asChild` 프롭 및 Radix `Slot` 도입으로 외부 컴포넌트(Link 등)와 스타일 유기적 결합 지원.
- **Header 리팩토링**:
  - `Bookmark`, `LogOut`, `Login (Link)`: 모두 `variant="ghost" size="icon"`으로 변경하여 일관된 디자인 시스템 적용 및 HTML 표준 준수.
- **의존성 추가**: `@radix-ui/react-slot` 패키지 설치.

## 3. 결과 (Result)

- 헤더 아이콘들의 디자인 및 인터랙션이 일관되게 적용됨.
- 개별 컴포넌트의 가독성 향상 및 전역 디자인 시스템 준수.

## 4. 검증 및 지표 (Verification & Metrics)

### 테스트 전략 및 항목

- [x] 브라우저 인터랙션 확인: 모든 아이콘 버튼의 호버 및 클릭 피드백 확인.
- [x] 커서 상태 확인: 마우스 호버 시 `pointer`로 변하는지 검증.
- [x] 린트 검사: 리팩토링 후 발생하는 린트 에러(asChild 관련 등)가 없는지 확인.
- [x] 브라우저 레코딩 및 스크린샷 증빙 (`header_icon_button_verification`)

- 린트 에러가 완전히 해결되어 안정적인 빌드 환경 유지.
