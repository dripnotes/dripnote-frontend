# [CHG-260416-10] BeanSearchBar IME 입력 및 버튼 타입 개선

## 1. 개요 (Overview)

한글과 같은 IME(Input Method Editor) 입력 시 엔터 키가 중복 처리되는 버그를 수정하고, UI 가독성 및 HTML 표준 준수를 위해 버튼 속성을 개선했습니다.

## 2. 상세 변경 내역 (Detailed Changes)

### 2.1 IME 입력 중 검색 방지

- **문제**: 한글 글자 조합 중 엔터를 누르면 조합 완료와 동시에 검색(onChange)이 실행되어 의도치 않은 중복 처리가 발생함.
- **해결**: `isComposing` 상태를 관리하여 `onCompositionStart`와 `onCompositionEnd` 사이의 `KeyDown(Enter)` 이벤트를 무시하도록 처리했습니다.

### 2.2 검색어 초기화 버튼 속성 개선

- **문제**: 버튼의 `type`이 명시되지 않아 폼 태그 내에 쓰일 경우 엔터 시 불필요한 폼 제출이 발생할 수 있음.
- **해결**: 초기화 버튼에 `type="button"`을 명시하여 폼 제출 동작과 분리했습니다.

## 3. 검증 결과 (Verification)

- [x] 한글 글자 조합(예: '커피') 중 엔터 입력 시 조합만 완료되고 검색은 1회만 트리거됨을 확인했습니다.
- [x] 초기화 버튼 클릭 시 `type="button"`이 올바르게 적용되어 폼 전송을 유발하지 않음을 확인했습니다.
