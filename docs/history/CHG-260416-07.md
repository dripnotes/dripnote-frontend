# [CHG-260416-07] BeanFilterDrawer 접근성 개선 및 하단 버튼 레이아웃 리뉴얼

## 1. 개요 (Overview)

모바일 필터 드로어의 웹 접근성(A11y)을 강화하고, 사용자가 필터 설정을 명시적으로 취소할 수 있도록 하단 버튼 레이아웃적 개편했습니다.

## 2. 상세 변경 내역 (Detailed Changes)

### 2.1 웹 접근성(ARIA) 강화

- **Dialog 역할 정의**: 드로어 컨테이너(`motion.div`)에 `role="dialog"` 및 `aria-modal="true"`를 추가하여 스크린 리더가 해당 영역을 독립된 대화상자로 인식하게 했습니다.
- **Labeling 연결**: 헤더 제목(`Filter`)에 고유 ID를 부여하고 드로어의 `aria-labelledby`와 연결하여 진입 시 컨텍스트를 명확히 했습니다.

### 2.2 하단 버튼 레이아웃 리뉴얼

- **취소 버튼 도입**: 헤더의 상단 X 버튼 대신 하단에 명시적인 '취소하기' 버튼을 추가하여 한 손 조작성을 높였습니다.
- **버튼 정렬 변경**: '적용하기'(좌) | '취소하기'(우) 순서로 배치하여 주요 동작을 우선적으로 노출했습니다.
- **시각적 가시성 개선**: '취소하기' 버튼에 테두리(`border-gray-200`)와 연한 배경색(`bg-gray-50`)을 적용하여 보조 버튼으로서의 시각적 구분을 명확히 했습니다.
- **반응형 비율**: 2:1 비율(`flex-[2]` vs `flex-1`)을 유지하여 적용하기 버튼의 주목도를 확보했습니다.

## 3. 검증 결과 (Verification)

- [x] 드로어 인스펙션 결과 `role="dialog"` 및 ARIA 속성 정상 반영 확인.
- [x] '취소하기' 버튼 클릭 시 `onClose` 정상 호출 및 드로어 닫힘 확인.
- [x] 버튼 순서 및 시각적 스타일(테두리, 색상) 기획 의도와 일치함 확인.
