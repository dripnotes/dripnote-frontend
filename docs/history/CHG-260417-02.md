# [CHG-260417-02] 필터 및 검색 설정 URL Query String 동기화 (2026-04-17)

## 1. 목적 (Purpose)

- 사용자가 선택한 필터 조건과 검색어를 URL과 실시간으로 동기화하여 상태 유지 및 공유 접근성을 향상시킵니다.
- 페이지 새로고침이나 뒤로 가기 시에도 사용자의 탐색 컨텍스트가 보존되도록 합니다.

## 2. 중간 과정 및 변경 사항 (Process & Deviations)

- **파라미터 명칭 통일**: 메인 페이지와 원두 페이지 간의 일관성을 위해 `aromaId`와 `aromas`로 혼용되던 파라미터를 `aromas`로 단일화했습니다.
- **상태의 중앙화 (URL SSOT)**: 로컬 `useState`로 관리되던 필터 상태를 제거하고, `useSearchParams`를 통해 URL로부터 도출되는 방식으로 구조를 변경했습니다.
- **표준화된 파라미터**: 아로마 명칭은 영문 ID를 사용하도록 설계하여 다국어 대응 및 URL 가독성을 확보했습니다.
- **유틸리티 강화**: `beans.ts`에 `encodeFiltersToParams` 및 `decodeParamsToFilters`를 추가하여 인코딩/디코딩 로직을 캡슐화했습니다.
- **사용자 경험 유지**: `router.push` 시 `scroll: false` 옵션을 사용하여 필터 적용 시 페이지 상단으로 튀는 현상을 방지했습니다.

## 3. 결과 (Result)

- 필터 적용 및 검색어 입력 시 URL이 `?q=...&aromas=...&bitterness=4` 형태로 실시간 업데이트됩니다.
- 공유된 링크를 통해 접속 시 즉시 동일한 필터링 결과가 노출됩니다.
- 초기화 버튼 클릭 시 URL 파라미터가 정리되어 기본 상태로 복구됩니다.

## 4. 검증 및 지표 (Verification & Metrics)

### 테스트 전략 및 항목

- **Browser Navigation Test**: 필터 적용 후 새로고침 시 UI 상태 유지 확인.
- **Search Sync Test**: 검색어 입력 후 Enter 시 URL 반영 및 결과 필터링 확인.
- **Deep Link Test**: 쿼리 파라미터가 포함된 URL로 직접 접속 시 필터링 결과 노출 확인.
