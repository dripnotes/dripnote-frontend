# 변경 이력 (Change History) - CHG-260408-10

## 1. 목적 (Purpose)

- **Layout Shift 해결**: `HeroSection`에서 인증 정보를 로드하는 동안(`isLoading: true`) 텍스트가 숨겨져 있다가 갑자기 나타나면서 발생하는 레이아웃 흔들림 현상을 방지합니다.
- **UI 라이브러리 고도화**: 재사용 가능한 `Skeleton` 컴포넌트를 구축하고, 공통 유틸리티(`cn`)를 추출하여 코드 유지보수성을 향상시킵니다.
- **Premium UX 제공**: 단순한 로딩 스피너 대신 브랜드 무드에 맞는 부드러운 펄스 애니메이션의 Skeleton을 제공합니다.

## 2. 과정 및 변경 사항 (Process & Deviations)

- **Infrastructure 및 리팩토링**:
  - `packages/ui-library/lib/utils.ts`: `clsx`와 `tailwind-merge`를 결합하여 클래스 오버라이드를 안전하게 처리하는 `cn` 함수를 별도 유틸리티로 분리했습니다.
  - `packages/ui-library/components/Button.tsx`: 내부의 `cn` 구현을 제거하고 공용 유틸리티에서 가져오도록 리팩토링했습니다.
- **컴포넌트 구현**:
  - `packages/ui-library/components/Skeleton.tsx`: `framer-motion`의 `animate={{ opacity: [0.5, 0.8, 0.5] }}`을 활용하여 은은하고 고급스러운 펄스 효과를 가진 스켈레톤 컴포넌트를 구현했습니다.
- **페이지 최적화**:
  - `apps/web/components/main/HeroSection.tsx`:
    - `h1` 타이틀 영역에 텍스트 길이와 유사한 폰트 크기 기반의 Skeleton(너비 48~80)을 적용했습니다.
    - `p` 설명 문구 영역에 2줄 형태의 Skeleton 구성을 추가하여 로딩 중에도 공간을 확보했습니다.

## 3. 결과 (Result)

- 페이지 초기 진입 및 새로고침 시 텍스트가 갑자기 나타나며 레이아웃이 밀리는 현상이 사라졌습니다.
- 로딩 단계부터 브랜드 아이덴티티가 반영된 정교한 placeholder UI를 제공함으로써 사용자 경험을 고도화했습니다.
- `ui-library`의 컴포넌트 작성 표준(cn 유틸리티 활용)을 확립했습니다.

## 4. 검증 및 지표 (Verification & Metrics)

- **타입 및 린트 검증**: `pnpm type-check` 및 `pnpm lint`를 통해 프로젝트 전체의 정합성을 확인했습니다.
- **Layout Shift 확인**: 브라우저 개발자 도구의 'Performance' 탭 및 네트워크 속도 제한(Throttling)을 통해 로딩 시 레이아웃 안정성을 시각적으로 검증했습니다.
- **동작 확인**:
  - `framer-motion` 기반의 펄스 애니메이션이 `Skeleton` 부착 부위에서 무한 반복되며 부드럽게 작동함을 확인했습니다.
  - 로딩 완료 후 실제 텍스트로 전환될 때 주변 요소(버튼 등)의 위치가 고정됨을 확인했습니다.
