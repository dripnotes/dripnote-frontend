# 변경 이력 (Change History) - CHG-260408-08

## 1. 목적 (Purpose)

- 인증 상태(`auth-utils`)에 따른 히어로 섹션 인사말 개인화 구현.
- 메인 페이지의 브랜드 보안 및 UX 향상을 위한 텍스트 선택 방지(`user-select: none`) 적용.
- 히어로 섹션 타이틀(`h1`)의 행간 조정을 통한 가독성 및 심미성 강화.

## 2. 과정 및 변경 사항 (Process & Deviations)

### [Infrastructure] 인증 하네스 구축

- `apps/web/lib/utils/user-mock.ts`: 백엔드 API 부재 시 가상 유저 정보를 제공하는 모킹 유틸리티 생성.
- `apps/web/hooks/use-auth.ts`: `localStorage`의 토큰 유무를 감지하여 유저 상태와 로딩 상태를 반환하는 커스텀 훅 구현.

### [UI/UX] 스타일링 및 레이아웃 조정

- `apps/web/app/globals.css`: `body` 태그에 `user-select: none` 속성을 추가하여 메인 페이지 내 텍스트 드래그 및 복사를 제한함.
- `apps/web/components/main/HeroSection.tsx`:
  - **행간 및 반응형 타이틀 조정**:
  - `h1` 태그의 첫 번째 줄과 두 번째 줄의 계층 구조를 분리하고, 두 번째 줄의 폰트 사이즈를 `text-[26px]`(반응형 모바일)~`5xl` 수준으로 상향하여 시각적 존재감과 가독성을 동시에 확보했습니다.
  - 투명도를 제거하고 선명한 흰색을 적용하여 가독성을 높였습니다.
  - 첫 번째 줄(`span`): 기존의 큰 폰트 사이즈 유지 (`text-4xl` ~ `md:text-6xl`).
  - 두 번째 줄(`span`): iPhone SE 등 좁은 해상도에서 잘리지 않도록 모바일 기본 사이즈를 미세 조정 (`text-[26px]` ~ `md:text-5xl`).
  - 알파값(투명도)을 적용하지 않고 순수 `white` 색상을 사용하여 명확하게 노출.
  - **무개행 처리**: 두 번째 줄에 `whitespace-nowrap`을 적용하여 어떠한 해상도에서도 개행이 발생하지 않도록 고정.
  - 설명 문구(`p`): `break-keep`(word-break: keep-all)을 적용하여 한글 단어가 중간에 끊기지 않고 띄어쓰기 단위로 자연스럽게 개행되도록 최적화.
  - `useAuth` 훅을 연동하여 인증 상태(`isAuthenticated`)에 따른 동적 메시지 분기 처리.
  - `isLoading` 상태 중에는 텍스트를 숨겨 하이드레이션 시의 텍스트 깜빡임 현상을 방지함.
