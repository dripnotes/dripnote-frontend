# [CHG-260420-17] 전역 레이아웃 표준화 및 모바일 UI 디테일 고도화 (2026-04-20)

## 1. 목적 (Purpose)

- `SectionContainer` 컴포넌트 도입을 통해 모든 페이지 섹션의 수평 여백과 중앙 정렬 로직을 표준화합니다.
- 모바일 환경에서의 가독성과 정보 밀도를 높이기 위해 메인 페이지 추천 섹션과 원두 카드 디자인을 최적화합니다.
- 공통 UI 컴포넌트(`RatingScale`)의 시인성을 개선하여 다양한 배경색에서 명확한 데이터를 전달합니다.

## 2. 중간 과정 및 변경 사항 (Process & Deviations)

- **전역 레이아웃 표준화 (`SectionContainer`)**:
  - `px-8` 수평 패딩과 `max-w-7xl` 중앙 정렬을 캡슐화한 `SectionContainer`를 신규 구현했습니다.
  - 원두 상세 페이지와 메인 페이지의 주요 섹션들을 이 컨테이너로 마이그레이션하여 레이아웃 정합성을 확보했습니다.
- **메인 페이지 '오늘의 추천 원두' 모바일 최적화**:
  - 모바일에서 노출되는 원두 개수를 2개에서 4개로 늘리고, **2x2 그리드** 배치를 적용하여 정보 탐색 효율을 높였습니다.
- **BeanCard 모바일 디자인 미세 조정**:
  - 2x2 그리드 배치에 맞춰 카드 내부 텍스트(원산지, 이름) 크기를 축소하고, 호버 오버레이의 아이콘 및 지표 레이아웃을 촘촘하게 재설계했습니다.
- **RatingScale 시인성 개선 (Placeholder)**:
  - 밝은 배경에서 보이지 않던 비활성 평점 단계의 색상을 `white/10`에서 `rgba(156, 163, 175, 0.2)`로 변경하여 라이트/다크 모드 모두에서 시인성을 확보했습니다.
- **추출 가이드(Brewing Guide) 레이아웃 조정**:
  - 모바일에서 **1열 배치(세로 쌓기)** 구조를 적용하여 정보 가독성을 확보하고, 텍스트 크기(`text-sm`) 및 하단 공백(`mb-10`)을 최적화했습니다.
- **목사 데이터 업데이트**:
  - `beans.ts` 내 원두 맛 지표(쓴맛, 단맛) 데이터를 실제 시각화 테스트를 위해 업데이트했습니다.

## 3. 결과 (Result)

- 서비스 전반의 섹션 레이라웃이 일치되어 정돈된 무드를 제공함.
- 모바일에서 한 화면에 더 많은 추천 정보를 제공하면서도 가독성을 유지함.
- 다양한 환경(라이트/다크)에서도 데이터 지표가 명확하게 노출됨.
