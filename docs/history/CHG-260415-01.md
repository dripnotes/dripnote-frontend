# [CHG-260415-01] 에이전트 규칙 체계 통합 및 스펙 컴포넌트 단위 마이그레이션 (2026-04-15)

## 1. 목적 (Purpose)

- **에이전트 규칙 파일 중복 제거**: `global.md`와 `process-rules.md`가 한국어 커밋, 문서 업데이트 의무 등 동일한 내용을 중복 서술하고 있어 단일 진실 공급원(SSOT)이 훼손되는 문제를 해결합니다.
- **규칙 공백 보완**: `process-rules.md`는 YAML frontmatter(`trigger`)가 없어 AI가 항시 로드하지 않을 수 있는 구조적 취약점이 있었습니다. 이를 `global.md`에 완전히 흡수하여 해결합니다.
- **컴포넌트 스펙 표준화**: 기존 스펙이 섹션 단위로만 분류되어 있어 컴포넌트별 Data Interface, UI 상태, 애니메이션 명세가 누락된 상태였습니다. `spec-guidelines.md` 표준 템플릿을 신규 제정하고 기존 스펙을 마이그레이션합니다.

## 2. 중간 과정 및 변경 사항 (Process & Deviations)

### 2-1. 규칙 파일 통합 (`global.md` ← `process-rules.md`)

`process-rules.md`의 고유 내용만 `global.md` Section 2에 흡수하고, 중복 항목(한국어 커밋, 문서 업데이트 의무)은 제거했습니다.

- **추가된 항목**:
  - 7단계 생명주기 준수 (Lifecycle Compliance)
  - 허가 단계 묵살 금지 (Approval Gate)
  - 외부 AI 리포트 대응 (AI Report Handling)
  - PLAN.md 의무 작성 (Section 3)
- **신규 규칙 추가** (사용자 요청):
  - 대규모 수정 사전 알림 (Large-scale Change Notice)
  - 다음 작업 제안 (Next Action Suggestions)
  - 컴포넌트 스펙 작성 가이드라인 준수 (Rule 28)

### 2-2. `spec-guidelines.md` 신규 제정

사용자가 제공한 5섹션 템플릿을 기반으로, 프로젝트 실정에 맞게 2개 섹션을 추가하여 7섹션 표준 템플릿을 확립했습니다.

- **원본 대비 추가된 섹션**:
  - `UI States` (상태 명세): Default / Loading / Error / Empty 상태 테이블
  - `Design Spec` (디자인 명세): framer-motion 애니메이션 스펙 + 반응형 분기점
- **기타 개선**: Props만 있던 Data Interface를 Props + State 테이블 + Events/Callbacks로 확장, DoD를 `[ ]` 체크리스트 형식으로 변경

### 2-3. 기존 스펙 마이그레이션

`login-page.md`: 5개 컴포넌트 명세화
`main-page.md`: 8개 컴포넌트 명세화

기존의 변경 사유(Context) 블록 및 API 연동 명세 섹션은 원형 보존했습니다.
